Supabase Sales Sync - Deployment Bundle

Contents expected in this folder (place before distributing to an outlet):
1) publish\  (your self-contained build output from `dotnet publish pos-sync-service -c Release -r win-x64 --self-contained true -o publish`)
2) install-service.ps1  (installer script)

How to prepare this bundle on your build machine:
- Run the publish command above; it will populate the publish/ folder.
- Copy scripts\install-service.ps1 from the repo into this folder (or keep the included copy if already present).
- Zip this SupabaseSalesSync folder to distribute to outlets.

How to install on an outlet machine (PowerShell as Administrator):
1) Unzip to, e.g., C:\Temp\SupabaseSalesSync
2) cd C:\Temp\SupabaseSalesSync
3) pwsh -File install-service.ps1 -PublishOutput "C:\Temp\SupabaseSalesSync\publish" -SkipPublish
   - Copies binaries to C:\Program Files\PosSyncService (default)
   - Creates config root at C:\Users\aliel\AppData\Local\XtZ (default) and seeds appsettings.json if missing
   - Installs/updates the service PosSupabaseSync and starts it

How to get PosDb.ConnectionString:

Service controls (Admin PowerShell):

- Uninstall: Stop-Service -Name PosSupabaseSync; sc delete PosSupabaseSync
