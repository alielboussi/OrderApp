Middleware install (outlet PC)
1) Copy the published folder to the outlet PC (e.g., installation/).
2) Edit appsettings.json in the config folder with POS DB, Outlet ID, Supabase URL/Key.
3) Open PowerShell as Administrator.
4) Run the installer script from the folder:
   pwsh -File .\scripts\install-service.ps1 -PublishOutput . -InstallPath "C:\Program Files\PosSyncService" -ConfigRoot "C:\Users\<you>\AppData\Local\XtZ"
5) Start the service (if not started) from Services or PowerShell.
6) Confirm it is running and syncing (service status/logs).

Stocktake procedure (outlet warehouse)
1) Ensure no sales/transfers are syncing during the count (pause/stop service if needed).
2) In the Stocktake app: select outlet + warehouse.
3) Start stocktake period (opening counts).
4) Enter opening counts and save.
5) Resume sales and transfers.
6) During the day, transfers/damages/sales update stock.
7) At closing time, pause/stop service to freeze movements.
8) Enter closing counts and close the period.
9) A new period is auto-opened and opening counts carry over from the closing counts.
10) Resume service and continue selling.
